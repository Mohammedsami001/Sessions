Lesson 3: Creating Our Workflow

Where do workflows live?

GitHub automatically looks for YAML files inside a very specific folder in your project: .github/workflows/.

If you put a YAML file there (e.g., ci.yml), GitHub will read it and run it.

Writing the Sessions CI Workflow

Here is the exact YAML we will use for our Next.js project. Let's break it down:

name: CI

# 1. The Trigger: When should this run?
on:
  push:
    branches: [ "main" ]
  pull_request:
    branches: [ "main" ]

# 2. The Jobs: What servers are we booting up?
jobs:
  build-and-test:
    runs-on: ubuntu-latest

    # 3. The Steps: What commands do we run?
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install dependencies
        run: npm ci

      - name: Run Unit Tests
        run: npx vitest run

      - name: Run Build
        run: npm run build

Understanding the Steps

  • uses: actions/checkout@v4: This is a pre-made action that downloads your repository's code onto the server.
  • uses: actions/setup-node@v4: This installs Node.js on the server so we can use npm.
  • run: npm ci: A strict version of npm install that is faster and better for automated servers.
