The Core Concept

GitHub Actions is a CI/CD (Continuous Integration / Continuous Deployment) platform built directly into GitHub. It allows you to automate your software development workflows.

Instead of manually running npm run build or npx vitest run every time you want to make sure your code works, GitHub Actions does it for you automatically on GitHub's servers whenever you push code or create a Pull Request.

Key Terminology

  • Workflow: An automated process made up of one or more jobs. Workflows are defined by a YAML file in the .github/workflows/ directory.
  • Event: A specific activity that triggers the workflow (e.g., a push or a pull_request).
  • Job: A set of steps that execute on the same runner (server).
  • Step: An individual task that can run commands (like npm install) or use predefined actions.
  • Runner: A server provided by GitHub (e.g., ubuntu-latest) that runs your workflow.
