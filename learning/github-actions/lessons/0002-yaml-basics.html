Lesson 2: YAML Basics

What is YAML?

YAML stands for "YAML Ain't Markup Language" (a recursive acronym). It is a highly readable data-serialization language. We use it for configuration files because it is incredibly easy for humans to read and write.

GitHub Actions uses YAML files to understand exactly what you want it to automate.

The 3 Golden Rules of YAML

1. Key-Value Pairs

Data is stored as a key followed by a colon, a space, and then the value.

name: Sessions CI
author: Sami

2. Indentation Matters (Like Python!)

You use spaces (never tabs) to nest data. Usually, we use 2 spaces for indentation. This tells YAML that the indented items belong to the key above them.

on:
  push:
    branches:
      - main

(In this example, push belongs to on, and branches belongs to push).

3. Lists use Dashes

When you want a list of items (an array), you put each item on a new line starting with a dash - and a space.

jobs:
  - build
  - test
  - deploy
